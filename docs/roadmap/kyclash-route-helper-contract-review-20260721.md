# KyClash macOS route-helper contract review

Status: approved and locked for S1.11–S1.12

Date: 2026-07-21

## Decision

KyClash uses one macOS 13+ `SMAppService` LaunchDaemon named
`net.kysion.kyclash.route-helper`. Its plist lives at
`Contents/Library/LaunchDaemons/net.kysion.kyclash.route-helper.plist`, and its
signed executable lives at
`Contents/Resources/kyclash-route-helper`. The plist uses only `BundleProgram`
and advertises the fixed Mach service `net.kysion.kyclash.route-helper`.

Registration, approval-state inspection, upgrade, and unregistration are owned
by the KyClash app. Registration never occurs at install time or merely because
the app starts. The production networking UI must explain the narrowly scoped
route capability before requesting registration. Actual registration and
system mutation evidence runs only in the authorized disposable macOS VM.

## Authentication

The daemon accepts an XPC connection only when macOS validates this fixed code
requirement:

```text
anchor apple generic and identifier "net.kysion.kyclash" and
certificate leaf[subject.OU] = "RQUQ8Y3S9H"
```

It additionally requires the connecting effective UID to be a real non-root
login user and records only the audit session ID and PID for the lifetime of
the connection. PID, UID, interface shape, or a caller-supplied bundle path is
never sufficient authentication. No alternate team, development ad-hoc
signature, inherited Mihomo service, or unsigned caller is accepted.

## Typed XPC surface

The Objective-C-visible protocol exposes exactly:

- `discover(reply)` with no caller-controlled command or path;
- `begin(request, reply)` with operation, sidecar instance, exact owned utun
  facts, policy revision, normalized private CIDRs, and lease identifier;
- `apply(request, reply)` for the exact previously begun transaction;
- `rollback(request, reply)` for the exact owned transaction;
- `recover(request, reply)` for a journal-owned transaction after restart;
- `heartbeat(request, reply)` for the exact live lease;
- `status(request, reply)` returning only bounded state and error codes.

Every request is an `NSSecureCoding` value with a fixed class allowlist. There
is no generic dictionary, JSON blob, shell text, command, executable, file
path, environment, arbitrary route argument, DNS value, or untyped selector.
Unknown fields, versions, enum values, duplicate CIDRs, invalid identifiers,
non-`utunN` interfaces, MTU other than 1420, zero revisions, and replayed or
mismatched lease/operation ownership fail closed before discovery or mutation.

## Lifecycle and fail-closed behavior

- One XPC connection owns at most one transaction and lease.
- XPC invalidation, heartbeat expiry, app death, helper restart, or explicit
  rollback enters the same idempotent rollback path.
- S1.11 discovery is read-only and all mutation methods return `not_ready`
  until the S1.12 journal/executor is linked.
- S1.12 persists intent before every route mutation. The helper accepts only
  canonical add/delete operations generated by its internal planner.
- Routes are rolled back before the sidecar may close the exact utun. DNS and
  default-route mutation are absent from the protocol and executable.

Changing the helper identity, Mach service, signing requirement, method set,
route ownership tuple, lease semantics, or allowing any generic command/path
input requires a new review record.

## Evidence gates

- Cross-platform serialization and validation tests cover every field,
  unknown data, replay/mismatch, injection-shaped values, and bounds.
- macOS compile tests inspect the plist, binary architecture, nested signature,
  Team ID, designated requirement, and XPC class allowlists.
- Injected helper tests prove invalidation and lease expiry use rollback.
- Registration, approval, route mutation, crash, restart, and uninstall tests
  execute only in the disposable VM.
